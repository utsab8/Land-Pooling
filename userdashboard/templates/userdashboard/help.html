Frequently Asked Questions (FAQ)

üîç
What file formats are supported?‚ñ∂
We currently support KML (.kml), CSV (.csv), and Shapefile (.shp, .shx, .dbf, etc. as a .zip archive) for upload and visualization. You can also export CSV data to Shapefile.
Why is my map not loading?‚ñ∂
Map loading issues can be due to several reasons:
  • Invalid Coordinates: Ensure your CSV or KML file contains valid and correctly formatted geographic coordinates (latitude, longitude).
  • Network Issues: Check your internet connection.
  • Server Error: If the problem persists, there might be a server-side issue. Please try again later or contact support.
  • Browser Compatibility: Ensure you are using a modern web browser.
How do I fix shapefile errors?‚ñ∂
If you encounter errors with shapefiles:
  • Missing Components: A shapefile consists of multiple files (.shp, .shx, .dbf, etc.). Ensure all required files are present and zipped together before uploading.
  • Invalid Geometry: Verify that the geometries (points, lines, polygons) in your data are correctly formed.
  • Projection Issues: While our system primarily uses WGS84 (EPSG:4326), ensure your data's projection is compatible or correctly transformed.
Can I export my data?‚ñ∂
Yes, you can export your processed CSV data as a Shapefile. We are continuously working to add more export formats in the future.

Troubleshooting Tips

Common Issue: Data not appearing in table

Solution: Ensure your CSV file has a header row. If using KML, verify that placemark names and descriptions are present. Check the browser console for any JavaScript errors.

Common Issue: Slow performance on map

Solution: For very large datasets, map rendering can be slow. Try filtering your data or zooming in to a specific area. Ensure your browser is up-to-date.
